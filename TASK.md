# TASK.md

## 이번 작업의 목표
생년월일 및 출생일시 입력부터 로그인 확인, 임시 결제, 임시 결과 조회까지 이어지는 최소 기능 플로우를 구현한다.  
이번 작업은 실제 사주 계산이나 실제 결제 연동이 아니라, 화면 이동과 인증 가드, 입력 검증이 올바르게 동작하는 상태를 만드는 데 목적이 있다.

## 참조 문서

- docs/spec/SPEC_saju_auth_payment_result.md

## 작업 범위 (이것만)
1. 메인화면 구현
   - 우상단 로그인/회원가입 버튼
   - 중앙 입력 영역: 생년월일, 출생일시, 제출 버튼
   - 입력값 누락 또는 형식 오류 시 오류 문구 표시

2. 로그인 연동
   - 제출 시 로그인 여부 확인
   - 미로그인 사용자는 로그인 화면으로 이동
   - 로그인 완료 시 임시 결제 화면으로 이동

3. 임시 결제 화면 구현
   - 실제 결제 없음 안내 문구 출력
   - 결제완료 버튼
   - 홈으로 버튼
   - 결제완료 클릭 시 임시 결과 화면 이동
   - 홈으로 클릭 시 메인화면 이동 및 입력값 초기화

4. 임시 결과 화면 구현
   - 결과 안내 메시지 출력
   - 홈으로 버튼 출력
   - 홈으로 클릭 시 메인화면 이동 및 입력값 초기화

5. 라우팅 가드 적용
   - 결제화면, 결과화면은 로그인 사용자만 접근 가능
   - 미로그인 직접 접근 시 로그인 화면으로 이동

## 완료 기준
- 메인화면에서 생년월일과 출생일시를 입력하고 제출할 수 있다.
- 입력값이 비어 있거나 잘못되면 제출되지 않고 오류 문구가 보인다.
- 미로그인 상태에서 제출하면 로그인 화면으로 이동한다.
- 로그인 완료 후 임시 결제 화면으로 정상 이동한다.
- 임시 결제 화면에서 결제완료 클릭 시 임시 결과 화면으로 이동한다.
- 임시 결제 화면과 임시 결과 화면의 홈으로 버튼이 메인화면으로 이동시키고 입력 상태를 초기화한다.
- 미로그인 사용자가 결제화면 또는 결과화면 URL로 직접 접근하면 로그인 화면으로 이동한다.
