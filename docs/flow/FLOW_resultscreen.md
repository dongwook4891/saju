# FLOW_resultscreen

## 메인 플로우
1. 사용자가 로그인 상태로 result 화면에 진입한다.
2. 시스템이 로그인 여부를 확인한다.
3. 비로그인 상태이면 로그인 화면으로 이동한다.
4. 로그인 상태이면 저장된 사주정보를 조회한다.
5. 사주정보가 없으면 입력 화면으로 이동한다.
6. 사주정보가 있으면 형식을 검증한다.
7. 형식이 올바르지 않으면 수정 화면으로 이동한다.
8. 형식이 유효하면 Asia/Seoul 기준 오늘 날짜를 계산한다.
9. 사용자 + 오늘 날짜 기준 저장 결과를 조회한다.
10. 저장 결과가 있고, `profile_updated_at <= result_generated_at` 이면 저장 결과를 표시한다.
11. 저장 결과가 없거나, `profile_updated_at > result_generated_at` 이면 Gemini 분석 요청을 보낸다.
12. Gemini가 구조화된 결과를 반환한다.
13. 시스템이 응답 형식과 필수 필드를 검증한다.
14. 검증 성공 시 결과를 저장한다.
15. 저장 완료 후 result 화면에 결과를 표시한다.
16. 사용자는 질문하기 버튼으로 채팅 화면으로 이동할 수 있다.
17. 사용자는 다시 분석하기 버튼으로 하루 1회 재생성할 수 있다.

## 저장 결과 재사용 플로우
1. 사용자가 같은 날 result 화면에 다시 진입한다.
2. 시스템이 당일 저장 결과를 조회한다.
3. 저장 결과가 있고 사주정보 수정 이력이 없으면 Gemini를 다시 호출하지 않는다.
4. 저장 결과를 그대로 표시한다.

## 사주정보 수정 후 플로우
1. 사용자가 사주정보를 수정한다.
2. 사주정보의 `profile_updated_at` 이 갱신된다.
3. 사용자가 result 화면에 다시 진입한다.
4. 시스템은 `profile_updated_at > result_generated_at` 여부를 확인한다.
5. 조건이 참이면 기존 결과를 최신값 기준 결과로 사용하지 않는다.
6. Gemini에 새 분석을 요청한다.
7. 새 결과를 저장하고 표시한다.

## 질문하기 버튼 플로우
1. 사용자가 질문하기 버튼을 누른다.
2. 시스템이 오늘 날짜 기준 result 결과 존재 여부를 확인한다.
3. 당일 result 결과가 없으면 채팅 화면으로 보내지 않고 result 생성 또는 result 화면 복귀를 처리한다.
4. 당일 result 결과가 있으면 오늘 날짜 기준 채팅방 존재 여부를 확인한다.
5. 오늘 채팅방이 있으면 기존 방으로 이동한다.
6. 오늘 채팅방이 없으면 새 채팅방을 생성한다.
7. 채팅방 생성 시 당일 result snapshot을 채팅 컨텍스트로 연결한다.
8. 사용자는 질문을 입력한다.
9. 시스템은 오늘 result snapshot과 최근 채팅 내역을 기준으로 AI에게 질문을 전달한다.
10. AI가 답변을 생성한다.
11. 사용자 메시지와 AI 메시지를 모두 저장한다.
12. 같은 날 재진입 시 기존 채팅 내역을 불러온다.

## 다시 분석하기 플로우
1. 사용자가 다시 분석하기 버튼을 누른다.
2. 시스템이 오늘 수동 재생성 사용 횟수를 확인한다.
3. 가능 횟수가 남아 있으면 Gemini를 다시 호출한다.
4. 새 결과를 저장하고 화면을 갱신한다.
5. 수동 재생성 사용 횟수를 차감한다.
6. 가능 횟수를 초과하면 제한 안내를 표시한다.

## 생성 실패 후 재시도 플로우
1. 최초 결과 생성 또는 다시 분석하기 요청 중 오류가 발생한다.
2. 시스템은 저장된 기존 결과 존재 여부를 확인한다.
3. 저장된 결과가 있으면 기존 결과를 표시한다.
4. 저장된 결과가 없으면 오류 문구와 재시도 버튼을 표시한다.
5. 사용자가 재시도 버튼을 누르면 Gemini 요청을 다시 보낸다.
6. 이 재시도는 수동 다시 분석하기 1회 제한에 포함하지 않는다.

## 예외 플로우

### 비로그인
1. 사용자가 result 화면에 접근한다.
2. 시스템이 비로그인 상태를 확인한다.
3. 로그인 화면으로 이동한다.

### 사주정보 없음
1. 시스템이 사주정보 조회를 시도한다.
2. 사주정보가 없음을 확인한다.
3. result 화면을 표시하지 않는다.
4. 입력 화면으로 이동한다.

### 사주정보 형식 오류
1. 시스템이 생년월일 또는 출생시간 형식을 검증한다.
2. 형식 오류가 있으면 Gemini 요청을 보내지 않는다.
3. 오류 안내를 표시할 수 있다.
4. 수정 화면으로 이동한다.

### Gemini 실패
1. Gemini 요청 중 오류 또는 타임아웃이 발생한다.
2. 당일 저장 결과가 있으면 기존 결과를 표시한다.
3. 저장 결과가 없으면 오류 문구와 재시도 버튼을 표시한다.

### Gemini 응답 형식 오류
1. Gemini 응답은 왔지만 JSON 형식이 잘못되었거나 필수 필드가 누락된다.
2. 결과를 저장하지 않는다.
3. 오류 문구와 재시도 버튼을 표시한다.

### 저장 실패
1. Gemini 응답은 정상이다.
2. DB 저장 중 오류가 발생한다.
3. 결과를 확정 표시하지 않는다.
4. 오류 문구와 재시도 버튼을 표시한다.

### 채팅 진입 실패
1. 사용자가 질문하기 버튼을 눌렀다.
2. 당일 결과가 없거나 채팅방 생성 중 오류가 발생했다.
3. 오류 안내를 표시한다.
4. result 화면으로 복귀하거나 재시도를 제공한다.

## 화면 상태
- loading_profile: 사주정보 조회 중
- checking_saved_result: 당일 저장 결과 조회 중
- generating_result: Gemini 분석 중
- ready: 결과 표시 가능
- empty_profile: 사주정보 없음
- invalid_profile: 사주정보 형식 오류
- generation_failed: 생성 실패
- save_failed: 저장 실패
- opening_chat: 채팅방 확인 또는 생성 중
- chat_ready: 채팅 사용 가능
- chat_failed: 채팅 진입 또는 응답 실패
