# FLOW_mainscreen

## 메인 플로우
1. 비로그인 사용자가 메인화면에서 생년월일·출생일시 입력
2. 입력값을 sessionStorage에 임시 저장
3. 사용자가 로그인 또는 회원가입 진행
4. 인증 성공
5. sessionStorage의 임시 입력값 존재 여부 확인
6. 생년월일, 출생 시간, 출생 분이 모두 있으면 현재 로그인 계정 기준으로 DB 저장
7. 저장 성공 후 sessionStorage 임시값 정리
8. 메인화면 진입 시 현재 로그인 계정의 저장값 조회
9. 저장값이 있으면 input value로 표시, 없으면 기본 플레이스홀더 표시

## 예외 플로우
- 인증 실패: sessionStorage 임시값 유지, 저장 없음
- 임시값 없음: 저장 없이 기본 플레이스홀더 표시
- 일부만 입력됨: 저장하지 않고 기본 플레이스홀더 표시
- DB 저장 실패: 로그인 유지, 내부 로그 기록, 사용자 에러 UI 없음, 기본 플레이스홀더 표시
- 조회 실패: 기본 플레이스홀더 표시
- 로그아웃 후 비로그인 진입: 기본 플레이스홀더 표시

## 조회 시점
- 메인화면 최초 로드 시 1회 조회
- 비로그인 상태에서 로그인 상태로 전환된 직후 1회 추가 조회 가능
- 주기적 재조회는 하지 않음

## 표시 규칙
- 로그인 사용자 + 저장값 있음: input value 표시
- 로그인 사용자 + 저장값 없음: 기본 플레이스홀더 표시
- 비로그인 사용자: 기본 플레이스홀더 표시
