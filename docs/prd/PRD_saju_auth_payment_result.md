# PRD_OOO

## 목적
사용자가 생년월일/출생일시를 입력한 뒤 로그인 여부를 확인하고, 임시 결제 후 임시 결과 화면까지 이동하는 최소 플로우를 구현한다.

## 사용자
- 비회원: 입력 후 로그인 필요
- 회원: 로그인 완료 후 결제/결과 진행

## 핵심 시나리오
1. 메인에서 생년월일, 출생일시 입력
2. 제출 시 로그인 상태 확인
3. 미로그인 시 로그인/회원가입 이동
4. 로그인 완료 시 임시 결제 화면 이동
5. 결제완료 클릭 시 임시 결과 화면 이동
6. 홈으로 클릭 시 메인으로 이동

## 예외/조건
- 입력값 누락 시 제출 불가, 오류 문구 표시
- 결과/결제 URL 직접 접근 시 로그인 확인 후 차단
- 결제는 실제 승인 없이 버튼만 동작
- 홈 이동 시 입력값 초기화
- 뒤로가기 시 보호 라우팅 유지

## 범위 제외
실제 결제, 실제 사주 계산, 결과 저장
